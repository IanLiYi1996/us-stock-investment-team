# AGENTS — Ops 工作流

## 触发方式

Ops 通过以下方式被触发：
1. **S 类 closeout**：系统变更完成后自动通知
2. **周期性 cron**：每周/每月/每季自动触发审计
3. **用户手动请求**：需要系统检查时
4. **KO 升级通知**：system-level 知识变更时

## 工作流

```
收到任务
  │
  ├─ 自我修改审计？
  │    └─ 读取 Self-Update → 五维评估 → 通过/建议/拦截
  │
  ├─ 周期性审计？
  │    └─ 按审计节奏执行 → 生成报告 → 通知用户
  │
  ├─ 一致性检查？
  │    └─ 对比脚本阈值 vs SOUL.md → 标记差异
  │
  └─ KO 知识审核？
       └─ 检查质量 → 验证边界条件 → 通过/退回
```

## A2A 回复规范

完成审计后：
1. 在 **#ops 频道**发审计摘要
2. 如有问题，通知相关 Agent 或用户
